# Статические данные вместо API

Фронтенд теперь полностью независим от Django бэкенда и работает со статическими JSON файлами.

## Структура данных

### Файлы данных:
- `src/data/services.json` - все услуги (19 услуг)
- `src/data/equipment.json` - техника (пустой, можно заполнить при необходимости)
- `src/data/categories.json` - категории (пустой, можно заполнить при необходимости)

### Как это работает:

1. **API модуль** (`src/services/api.ts`) теперь загружает данные из JSON файлов вместо HTTP запросов
2. Все компоненты продолжают использовать те же методы API (`servicesApi.getAll()`, и т.д.)
3. Данные возвращаются в том же формате, что и раньше (для обратной совместимости)

## Преимущества:

✅ Фронтенд работает без Django бэкенда
✅ Быстрая загрузка (нет сетевых запросов)
✅ Можно деплоить как статический сайт
✅ Все данные в одном месте - легко редактировать

## Редактирование данных:

Чтобы изменить информацию об услугах, просто отредактируйте файл `src/data/services.json`.

**Формат данных:**
```json
{
  "count": 19,
  "results": [
    {
      "id": 1,
      "name": "Название услуги",
      "slug": "slug-uslugi",
      "short_description": "Краткое описание",
      "phone": "+7 988 830-77-77",
      "whatsapp_link": "https://wa.me/...",
      "call_link": "tel:+79888307777",
      ...
    }
  ]
}
```

## Примечания:

- Альтернативные slug'и (например, `kopka-transhej` → `kopka-transej`) обрабатываются в компонентах через маппинг `servicePageMap`
- Изображения услуг загружаются из папки `src/images/images-imag/` через маппинг `serviceImageMap`
- Все 19 услуг уже настроены и готовы к использованию
